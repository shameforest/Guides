§1
Pipeline-Konzept und Skelett-Wahl

Animation ist universal, Skelett ist spezifisch.

Die Animationen aus 01-09 sind alle auf dem UE5-Manny-Skelett gebaut. Wenn dein Projekt einen Custom-Charakter braucht — anderer Körperbau, andere Proportionen, andere Personality — bleibt die Animation die gleiche, aber das Skelett ist anders. Diese Brochüre zeigt, wie du den Pipeline-Bruch dazwischen sauber überbrückst.

Workflow: dein Custom-Charakter ist in Blender mit Auto-Rig Pro gerigt — Industrie-Standard für Blender-Rigging mit Control-Rig und Deform-Rig. Auto-Rig Pro produziert ein sehr komplexes Control-Rig (gut für Blender-Animation) und ein einfacheres Deform-Rig (für Game-Engines). Cascadeur braucht das Deform-Rig, aber „direkt" exportiert kommt das Control-Rig mit. Lösung: Game Rig Tools (CG Dive, kostenlos auf Gumroad) — ein Blender-Addon, das ein simplifiziertes Game-Rig generiert, ohne Control-Constraints.

In Cascadeur kommt der Charakter mit dem Game-Rig an. Quick Rigging Tool wird manuell verwendet, um Cascadeurs eigene Controller dem Custom-Skelett zuzuordnen. Template speichern für Wiederverwendung. Dann: bestehende Animation aus 02-09 importieren und auf den Custom-Charakter übertragen — Retargeting (Pro) oder Copy/Paste Interval bei identischen Bone-Namen.

Warum diese Pipeline existiert

UE5-Manny ist Standard und einfach. Aber Game-Charaktere sind selten Standard — sie haben individuelle Proportionen, Stile, manchmal nicht-humanoide Elemente (Schwanz, zusätzliche Glieder, Stilisierung). Auto-Rig Pro ist in Blender die Standardlösung für Custom-Rigging. Cascadeur kann mit beliebigen humanoiden Skeletten arbeiten, aber braucht eine sauber strukturierte Deform-Rig-Variante. Game Rig Tools schließt diese Lücke.

Es gibt eine historische Pipeline-Frage, die seit den 1990ern offen ist: sollte Animation auf dem Control-Rig (animator-freundlich) oder dem Deform-Rig (engine-freundlich) gebaut werden? Maya hat das Problem mit „HumanIK" gelöst, Blender mit Rigify, jetzt erweitert um Auto-Rig Pro und Game Rig Tools. Cascadeur ist eine spätere Generation — sein Quick Rigging Tool baut ein neues Cascadeur-Control-Rig auf jedem Deform-Skelett. Das gibt drei Rig-Layer: dein originales Blender Control-Rig (für Modeling und Setup), Auto-Rig Pro's Deform-Rig (für Game-Engine), und Cascadeur's AutoPosing-Rig (für die Animation). Klingt komplex, ist es auch — aber jedes Layer hat seinen Sinn.
§2
Phase A · Blender-Prep

Auto-Rig Pro → Game Rig Tools.

A

Blender-Prep

Auto-Rig Pro · Game Rig Tools · Connect Bones · FBX-Export

Charakter in Blender mit Auto-Rig Pro fertig. Game Rig Tools-Addon installieren. Game Rig generieren. Mute Control Rig. Bones connecten. FBX exportieren. Vier Schritte.

Zwei Rig-Layer in Blender: Control-Rig für Posing, Deform-Rig für Export.
01 Charakter mit Auto-Rig Pro in Blender

Voraussetzung: Custom-Charakter ist in Blender mit Auto-Rig Pro fertig geriggt. Das umfasst:

  • Mesh mit allen Body-Parts (Torso, Arme, Beine, Kopf, Hände, Finger)
  • Auto-Rig Pro Control-Rig mit IK/FK-Switches, Constraints, Custom-Bone-Shapes
  • Skinning/Weight-Painting für alle Bones
  • Optional: Auto-Rig Pro's eingebauter „Smart" für automatisches Bone-Placement

Dies ist der Start-Punkt der Brochüre — der Rigging-Prozess selbst ist nicht Teil der Cascadeur-Reihe, sondern ein eigenes Pipeline-Thema. Auto-Rig Pro hat hervorragende eigene Doku und Tutorials für diesen Vorlauf.

Wichtig: das Skelett enthält jetzt ein Control-Rig (komplex, mit Constraints) und ein Deform-Rig (simpel, nur Bones). Auto-Rig Pro speichert beide in einer Armature. Für den Cascadeur-Export ist das Deform-Rig relevant — aber direkt exportiert kommt das Control-Rig auch mit, was Probleme verursacht.

Auto-Rig Pro vs Rigify

Beide Blender-Rigs haben ähnliche Probleme im Cascadeur-Export. Game Rig Tools ist primär für Rigify entwickelt, funktioniert aber auch mit Auto-Rig Pro (mit kleinen manuellen Anpassungen). Falls dein Charakter mit Rigify gerigt ist, gilt der gleiche Workflow — das Setup ist identisch. Bei Auto-Rig Pro hat man manchmal mehr custom Bones (Twist Bones, Extra Bones), die im Game Rig Tools nicht automatisch verarbeitet werden — manuelle Connect-Schritte sind dann nötig (Schritt 03).

Game Rig Tools (CG Dive) Installation. Kostenlos via Gumroad mit Preis-Schieber auf 0.
02 Game Rig Tools Addon installieren und Game Rig generieren

Game Rig Tools von CG Dive auf cgdive.com/comic-tools-blender-add-on oder direkt auf Gumroad. Kostenlos — bei Gumroad den Preis-Slider auf 0 setzen (optional spenden, um den Entwickler zu unterstützen). „Core Module" auswählen, nicht „Unreal Module" (das ist separat und nicht nötig für unseren Workflow).

Installation:

  1. Blender starten. Edit → Preferences
  2. Tab „Add-ons" → Install from Disk → das Game Rig Tools ZIP wählen
  3. Addon aktivieren (Checkbox)

Game Rig generieren:

  1. In den Pose Mode wechseln (nicht Object Mode — das kann zum Crash führen)
  2. Control-Rig selektieren
  3. Im Game Rig Tools-Panel: Generate Game Rig
  4. Name vergeben (z.B. „cascadeur") und bestätigen

Ergebnis: ein neues Skelett im Outliner, das das Deform-Rig isoliert (ohne Control-Constraints), bereit für Engine-Export.

Pose Mode:Ctrl+Tab
Connect-Bones-Schema. Face/Shoulders mit Keep Offset, Arms/Legs connected. Twist Bones NICHT.
03 Connect Bones und Mute Control Rig

Das Game Rig hat noch Hierarchie-Probleme. Cascadeur erwartet eine bestimmte Bone-Hierarchie für automatisches Quick Rigging Tool-Erkennen. Standardmäßig sind die Game-Rig-Bones unconnected, was problematisch ist.

Vorgehen aus dem Game-Rig-Tools-Tutorial:

  1. In den Edit Mode wechseln (im Game-Rig)
  2. Face-Bones selektieren (Mirror-Tool für die andere Seite), dann Ctrl+P → Keep Offset
  3. Shoulders selektieren, Ctrl+P → Keep Offset
  4. Arms (ohne Twist Bones!) selektieren, Ctrl+P → Connected
  5. Forearm an Upper-Arm connecten, Ctrl+P → Connected
  6. Legs an dev_spine (oder den entsprechenden Spine-Bone) connecten, Ctrl+P → Connected

Twist Bones NICHT connecten — sie sind extra Bones für realistisches Skinning, die mit eigener Rotation arbeiten. Wenn man sie connectet, gibt es Probleme im AutoPosing für Finger.

Vor dem FBX-Export: im Game-Rig-Tools-Panel Mute Control Rig drücken. Damit wird das Original-Control-Rig deaktiviert — sonst überschreibt es alles im Game-Rig.

Mute Control Rig nicht vergessen

Falls Mute Control Rig nicht aktiviert ist, kann das Control-Rig im Hintergrund weiterhin Bones überschreiben. Beim FBX-Export wäre dann immer noch das Control-Rig im Export — Cascadeur-Import würde scheitern oder ein verwirrendes Skelett zeigen. Erkennbar daran, dass nach Generate Game Rig die Game-Rig-Bones nicht manuell bewegbar sind — sie werden vom Control-Rig dominiert. Mute → bewegbar.

FBX-Export aus Blender. Nur Game-Rig auswählen, Y-Forward, Z-Up.
04 FBX-Export aus Blender

File → Export → FBX. Wichtige Settings für Cascadeur-Pipeline:

  • Selected Objects: ON — nur das Game-Rig und das Mesh werden exportiert (vorher selektieren)
  • Apply Scalings: FBX All (Standard)
  • Forward Axis: -Y oder Y (je nach Blender-Default)
  • Up Axis: Z (für Cascadeur und UE5 obligatorisch)
  • Mesh: Apply Modifiers: ON (damit alle Modifier gebakt sind)
  • Armature: Primary Bone Axis: Y
  • Use Armature FBX Node Type: Null (kompatibler mit Cascadeur)
  • Add Leaf Bones: OFF (sonst entstehen leere End-Bones, die im Cascadeur stören)

Dateiname: character-custom.fbx.

Add Leaf Bones-Default

Blender's FBX-Export aktiviert standardmäßig „Add Leaf Bones". Das fügt zusätzliche End-Bones an jedes Finger-Ende und Bone-Tail an. Im Cascadeur ist das verwirrend — die Quick Rigging Tool sieht mehr Bones als der Charakter eigentlich hat. Deaktivieren spart spätere Verwirrung.

§3
Phase B · Cascadeur-Setup

Quick Rigging Tool manuell.

B

Cascadeur-Setup

FBX-Import · Quick Rigging Tool manuell · Template speichern

Import, manuelle Joint-Zuordnung, Generate Rig, Template-Speichern für spätere Wiederverwendung. Drei Schritte.

FBX-Import als Model. Bei Custom-Skelett-Strukturen ist manuelle Joint-Zuordnung nötig.
05 FBX in Cascadeur importieren als Model

Cascadeur starten, neue Scene. File → Import → FBX. Wichtig: Model-Preset wählen, nicht „Animation only". Bei Custom-Charakter wollen wir den Charakter selbst importieren, nicht nur die Animation-Daten.

Nach dem Import erscheint ein Popup „Switch to Rig Mode?" — mit Yes bestätigen. Cascadeur erkennt den Charakter und versucht, das Skelett-Template zu identifizieren.

Bei einem standardisierten Skelett (Mixamo, UE5) erkennt Cascadeur das automatisch und das Quick Rigging Tool füllt sich selbst aus. Bei Custom-Charakter ist das meist nicht der Fall — die Bone-Namen aus Blender (Auto-Rig Pro / Game Rig Tools) entsprechen keinem Cascadeur-Template.

Daher: manueller Quick-Rigging-Tool-Prozess (Schritt 06).

Template-Erkennung verbessern

Game Rig Tools versucht standardisierte Bone-Namen zu vergeben (mixamorig-style oder UE5-style, je nach Setting). Bei perfekter Standard-Konformität würde Cascadeur das Template automatisch erkennen. In der Praxis weichen Custom-Charaktere meist ab — extra Bones, andere Reihenfolge — sodass manueller Quick Rigging Tool nötig ist.

Quick Rigging Tool: jeder Schematik-Punkt wird einem Custom-Bone zugeordnet.
06 Quick Rigging Tool manuell ausfüllen

Im Rig Mode öffnet sich das Quick Rigging Tool-Schema. Es zeigt eine schematische Humanoid-Figur mit Punkten an Standard-Joint-Positionen (Head, Neck, Spine_01, Pelvis, L/R_Shoulder, L/R_Hip, etc.).

Vorgehen aus dem Mocap-Cleanup-Course:

  1. Custom-Skelett im Outliner anzeigen (alle Bones sichtbar)
  2. Jeden Schematik-Punkt anklicken (z.B. „Head")
  3. Den entsprechenden Bone im Custom-Skelett selektieren (z.B. „head_def" oder wie auch immer Auto-Rig Pro ihn benannt hat)
  4. Connect-Button drücken oder Drag-and-Drop in den Schematik-Punkt
  5. Für alle Punkte wiederholen — Head, Neck, Spine, Pelvis, Schultern, Arme, Beine, Hände (mindestens)
  6. Optional: Finger-Bones, Twist-Bones, Extra-Bones (Schwanz, etc.) in den entsprechenden Tabs

Mirror-Tool ist hilfreich: einmal die linke Seite ausfüllen, dann Mirror, Cascadeur generiert die rechte Seite automatisch (basierend auf Bone-Naming-Convention, z.B. L_ ↔ R_).

Bei vollständiger Zuordnung: Generate Rig-Button. Cascadeur baut sein internes Control-Rig auf dem Custom-Skelett.

Bone-Naming-Konvention

Wenn die Bone-Namen nicht der L/R-Konvention folgen (z.B. „Arm_left" statt „L_Arm"), funktioniert Mirror-Auto-Fill nicht. Dann jede Seite manuell ausfüllen. Vor dem Cascadeur-Import in Blender die Bone-Namen prüfen und ggf. standardisieren — spart Zeit bei jedem Re-Import.

Template speichern: gleiche Custom-Charakter-Skelette werden später automatisch erkannt.
07 Template speichern und Scene-Save

Nach erfolgreichem Generate Rig: das Skelett-Template speichern, damit bei zukünftigen Imports desselben Charakters (oder ähnlich aufgebauter Charaktere) Cascadeur den Rig automatisch erkennt.

Vorgehen:

  1. Im Quick Rigging Tool: Save as Template
  2. Name vergeben (z.B. „CustomCharacter_v1")
  3. Template wird in Cascadeur's Template-Bibliothek gespeichert

Beim nächsten Import desselben Custom-Charakters (oder eines mit identischem Skelett aber anderem Mesh — z.B. ein Outfit-Wechsel) erkennt Cascadeur das Template automatisch und das Rig wird sofort generiert.

Scene speichern als projekt-10-character-rigged.casc. Diese Datei ist jetzt der „leere Charakter" — Custom-Mesh + Custom-Skelett + Cascadeur-Control-Rig. Animation kommt im nächsten Schritt dazu.

Reusable Setup als Pipeline-Beschleuniger

Wenn dein Workflow mehrere Custom-Charaktere mit demselben Skelett verwendet (typisch bei Outfit-Varianten oder Body-Skin-Wechsel innerhalb einer Charakter-Familie), spart das gespeicherte Template enorm Zeit. Erstmals Setup: 15-30 Minuten. Folge-Imports: 30 Sekunden.

§4
Phase C · Animation laden

Source-Animation und Skelett-Vergleich.

C

Animation laden

Source aus 02-09 · Skelett-Vergleich · Retargeting-Wahl

Eine fertige Animation aus den vorigen Projekten laden. Skelette vergleichen — gleicher Bone-Name = Copy/Paste, anders = Retargeting. Drei Schritte.

Zwei-Tab-Workflow: Custom-Charakter und Source-Animation parallel offen.
08 Source-Animation in zweitem Tab laden

Cascadeur unterstützt mehrere Scenes parallel in verschiedenen Tabs. Das ist der Standard-Workflow für Animation-Übertragung.

Vorgehen:

  1. Tab 1: Custom-Charakter-Scene aus Phase B (mit leerem Rig, ohne Animation)
  2. Tab 2: eine fertige Animation aus den vorigen Projekten — z.B. manny-walk-v01.fbx aus Projekt 03 oder eine andere Animation aus 02-09
  3. In Tab 2 die Animation laden — entweder die .casc-Datei öffnen oder die FBX-Datei importieren

Jetzt hast du beide Scenes gleichzeitig: links der leere Custom-Charakter, rechts der animierte UE5-Manny.

Welche Animation als Source wählen

Idle (Projekt 01), Walk (03), Stand-Up (05), Gestik (08) sind „universal" und gut für Custom-Charakter-Tests. Sprung (07) ist anspruchsvoller — die Hindernis-Höhe muss zum Custom-Charakter passen, sonst springt er drüber oder dagegen. Mocap-Animation (09) ist möglich, aber die Pose-Qualität kann beim Retargeting auf andere Proportionen leiden. Anfangen mit Walk oder Idle — wenn die funktionieren, klappt der Rest auch.

Skelett-Vergleich-Entscheidungsbaum. Identische Namen → Copy/Paste. Anders → Retargeting.
09 Skelette vergleichen und Übertragungs-Methode wählen

Cascadeur unterscheidet zwei Fälle für Animation-Übertragung zwischen Scenes:

  • Bone-Namen identisch: Cascadeur erkennt es als „gleiches Skelett". Copy Interval + Paste Interval funktioniert direkt. Schnell, einfach, funktioniert ohne Pro-Version
  • Bone-Namen unterschiedlich: Cascadeur betrachtet es als „anderes Skelett", auch wenn die Struktur ähnlich ist. Retargeting Copy/Paste nötig (Cascadeur Pro Feature)

Praxis: dein Custom-Charakter aus Auto-Rig Pro hat sehr wahrscheinlich andere Bone-Namen als UE5-Manny — anders benannte Spine-Bones, eigene Naming-Conventions. Daher: meist Retargeting-Fall.

Test: in Tab 2 (Source) alle Joints selektieren, Edit → Copy Interval. In Tab 1 (Target) alle Joints selektieren, Edit → Paste Interval. Wenn eine Fehlermeldung „Names of guides or object to copy and objects to paste don't match" erscheint: Retargeting-Fall.

„Don't match"-Fehler beim Paste Interval

Standard-Fehlermeldung bei unterschiedlichen Bone-Namen. Bedeutet nicht „kaputt", sondern „nicht für direkten Copy/Paste geeignet". Lösung: Retargeting (Pro) oder UE5-Retargeting (kostenlos, siehe Schritt 11).

Target-Frame-Range muss zur Source passen. Erweitern, wenn nötig.
10 Target-Frame-Range erweitern

Wichtige Vorbereitung vor Copy/Paste oder Retargeting: die Target-Scene (Custom-Charakter, Tab 1) muss mindestens so viele Frames haben wie die Source-Animation.

Standardmäßig hat eine neue Cascadeur-Scene 30-90 Frames. Wenn die Source 200 Frames hat (z.B. eine Walk-Animation aus Projekt 03), würde Paste Interval nur die ersten 90 Frames übertragen.

Vorgehen:

  1. In Tab 1 (Custom-Charakter): auf der Timeline navigieren
  2. Right-Click auf den Time-Slider → Set Animation Length oder ähnlich (UI variiert leicht zwischen Cascadeur-Versionen)
  3. End-Frame entsprechend setzen (z.B. 200 oder 300 — gleich oder mehr als Source)
  4. Optional: bei sehr langen Source-Animationen den Quick-Expand-Button im Timeline-Panel nutzen

Frame-Range zu klein → Animation wird abgeschnitten. Frame-Range zu groß → kein Problem, die extra Frames bleiben leer.

§5
Phase D · Übertragung

UE5-Retargeting bevorzugt.

D

Übertragung

UE5-IK-Retargeter · Cascadeur Pro · Copy/Paste Interval

UE5-Retargeting ist sauberer und schneller (Deform-Rig → Deform-Rig). Cascadeur-Retargeting als Backup. Beide Wege gezeigt. Drei Schritte.

UE5 IK Retargeter: direkter Deform-Rig-Pfad. Sauberer als Cascadeur-Retargeting.
11 UE5-Retargeting (bevorzugter Pfad)

Aus dem Unreal-Retargeting-Tutorial: UE5's IK Retargeter ist deutlich besser als Cascadeur's eingebautes Retargeting. Grund: UE5 retargetet direkt Deform-Rig → Deform-Rig, ohne Control-Rig-Proxy-Zwischenstufe. Cascadeur retargetet Control-Rig → Control-Rig (mit Deform-Rig als Proxy dazwischen) — länger und unsauberer.

Vorgehen:

  1. Source-Animation (UE5-Manny mit Animation) als FBX aus Cascadeur exportieren
  2. In UE5 importieren — Skeletal Mesh, Animation Sequence, Physics Asset, Skeleton werden angelegt
  3. Custom-Charakter (aus Cascadeur Tab 1, ohne Animation) ebenfalls als FBX exportieren und in UE5 importieren
  4. In UE5 Content Browser: Rechtsklick auf die Source-Animation → Retarget Animations
  5. Source Skeleton: UE5-Manny. Target Skeleton: Custom-Charakter
  6. Bone-Mapping prüfen — meist erkennt UE5 viele Bones automatisch. Bei Abweichungen manuell mappen (Pelvis → Pelvis, Spine_01 → Spine_01, etc.)
  7. Export Animations → Custom-Charakter hat jetzt eine retargetete Animation Sequence

Optional: die retargetete Animation als FBX exportieren aus UE5 zurück nach Cascadeur — falls weitere Cascadeur-Polish nötig ist (z.B. AutoPhysics auf der retargeteten Version).

UE5-Retargeting als Pipeline-Standard

Aus dem Briefing: bei Export-Fragen UE5-Retargeting bevorzugen statt Cascadeur-Retargeting. Diese Brochüre folgt dem Standard. UE5-Retargeting funktioniert mit Free-Lizenz; Cascadeur-Retargeting braucht Pro. Auch wenn du Cascadeur Pro hast: UE5-Retargeting ist trotzdem oft die bessere Wahl wegen der Deform-Rig-Direkt-Übertragung.

Cascadeur-Pro-Retargeting als Backup. Beide Charaktere brauchen Cascadeur-Rigs.
12 Cascadeur-Pro-Retargeting als Backup

Wenn UE5-Retargeting aus irgendwelchen Gründen nicht funktioniert (zu exotisches Skelett, UE5 nicht verfügbar, oder Workflow-Präferenz): Cascadeur-Pro-Retargeting als Backup.

Voraussetzung: beide Charaktere brauchen Cascadeur-Rigs (Generate Rig erfolgreich). Bei dieser Brochüre ist das der Fall — der Custom-Charakter hat das Cascadeur-Rig aus Phase B, der UE5-Manny-Source ebenfalls (aus den vorigen Projekten).

Vorgehen:

  1. In Tab 2 (Source mit UE5-Manny-Animation): Select All in der Scene (Click auf den Charakter-Namen wählt alle Frames)
  2. Edit → Retargeting Copy
  3. In Tab 1 (Custom-Charakter) wechseln
  4. Frame-Range erweitern, wenn nötig (siehe Schritt 10)
  5. Select All in der Custom-Charakter-Scene
  6. Edit → Retargeting Paste

Cascadeur überträgt die Animation. Erwartete Probleme: kleine Pose-Abweichungen wegen unterschiedlicher Bone-Längen zwischen Source und Target. Bei stark unterschiedlichen Proportionen (z.B. Source ist Standard-Mannequin 180 cm, Target ist klein und stämmig) kann die Animation seltsam aussehen — die Bewegungs-Geste bleibt, aber die Posen passen nicht zum Charakter.

Pose-Verzerrungen bei stark unterschiedlichen Proportionen

Retargeting (egal ob Cascadeur oder UE5) kann nicht zaubern. Wenn der Source-Charakter 180 cm groß und schlank ist und der Target 150 cm und stämmig, wird die Animation auf dem Target nicht perfekt aussehen. Lösung: Posen nach dem Retargeting nochmal mit Cascadeur-Werkzeugen polishen — Hände an erwartete Positionen ziehen, Schritt-Längen anpassen, Knie-Beugen verfeinern. Das ist ein eigener Polish-Pass (Schritt 14).

Copy/Paste Interval: schnellster Pfad bei identischen Skeletten.
13 Copy/Paste Interval bei identischen Skeletten

Wenn das Custom-Skelett zufällig die gleichen Bone-Namen wie UE5-Manny hat (z.B. weil du in Auto-Rig Pro das UE5-Manny-Naming verwendet hast): direkter Copy/Paste-Workflow ohne Retargeting.

Aus dem Retargeting-Tutorial belegt:

  1. In Source-Scene: zum Joint-Mode wechseln (statt AutoPosing oder Point Controller)
  2. Joints selektieren (Strg+A oder Click auf Joint-Root)
  3. Edit → Copy Interval mit dem ganzen Range
  4. Target-Scene wechseln, Frame-Range erweitern wenn nötig
  5. In Target zum Joint-Mode wechseln, alle Joints selektieren
  6. Edit → Paste Interval

Funktioniert direkt — Animation ist auf dem Custom-Charakter, ohne Retargeting-Schritte. Schnellster Weg, wenn die Skelette kompatibel sind.

Finger via Copy/Paste Interval — Achtung

Finger-Animation kann via Copy/Paste Interval Probleme geben — selbst bei identischen Skelett-Namen werden Finger-Bones manchmal nicht übertragen. Für Finger-Animation auf Custom-Charakter: separater Schritt mit Finger-AutoPosing oder dediziertes Retargeting (Cascadeur Pro hat Retargeting auch für Finger).

§6
Phase E · Polish + Track-Update

Pose-Anpassung und Custom-Tracks.

E

Polish

Pose-Anpassung · Track-Hierarchie für Custom-Charakter

Retargeting-Artefakte korrigieren. Custom Track Hierarchy für den neuen Charakter setzen. Zwei Schritte.

Typische Retargeting-Artefakte. Foot-Drift und Arm-Reichweite häufige Pflicht-Fixes.
14 Retargeting-Artefakte diagnostizieren und polishen

Nach Retargeting (entweder UE5 oder Cascadeur) hat die Animation auf dem Custom-Charakter typische Artefakte, ähnlich zu Mocap-Cleanup (Projekt 09):

  • Foot-Position-Drift: Bei Proportions-Unterschied (Custom-Charakter kürzere/längere Beine) landen die Füße nicht mehr exakt am Boden. → Fulcrum Motion Cleaning aus Projekt 03
  • Arm-Reichweite zu lang/zu kurz: Wenn Manny eine Geste mit Arm-Reichweite X gemacht hat und der Custom-Charakter andere Arm-Länge hat, kann die Hand-End-Position verschoben sein. → Manuelle Pose-Korrektur, oder Trajectory Tangents
  • Spine-Twists: Selten bei kleinen Proportions-Unterschieden, häufiger bei sehr stylisierten Charakteren. → Selektives Re-Inbetweening
  • Finger-Posen-Verlust: Bei Retargeting werden Finger oft nicht oder schlecht übertragen. → Finger-AutoPosing neu anwenden, idealerweise Shift+Z über die ganze Sequenz
  • Body-Trajectory und Timing: bleibt meist gut. Retargeting respektiert die Bewegungs-Logik

Die Polish-Werkzeuge sind alle bekannt aus den vorigen Projekten — Fulcrum Motion Cleaning (03/09), Trajectory Tangents (06), Finger-AutoPosing (08), Selektives Re-Inbetweening (09). Hier kommen sie kombiniert zum Einsatz.

Es gibt ein Pipeline-Phänomen, das in der 3D-Branche „Charakter-Schock" heißt: wenn eine etablierte Animation auf einen neuen Charakter retargetet wird, sieht sie immer leicht „off" aus. Nicht weil das Retargeting schlecht ist, sondern weil die Animation für eine bestimmte Körperlichkeit gebaut wurde. Pixar-Animatoren machen daher fast nie pure Retargeting — sie polishen jede Animation pro Charakter neu, weil jeder Charakter „seine eigene Bewegung" haben soll. Für Game-Pipelines mit dutzenden NPCs ist das nicht möglich; man retargetet und akzeptiert kleine Inkonsistenzen. Für Hero-Charaktere in Cutscenes lohnt sich der Pro-Pixar-Polish.
Track-Hierarchie für Custom-Charakter. Standard-Groups plus Custom-spezifische Tracks.
15 Custom Track Hierarchy für den neuen Charakter

Nach Generate Rig hat Cascadeur eine Default-Track-Struktur für das Custom-Skelett angelegt. Diese ist oft minimal — alle Body-Parts in einem Track, ohne Trennung. Für effizienten Animator-Workflow sollte sie umgebaut werden.

Wie in Projekt 03 / 08 / 09 — Custom Track Hierarchy anlegen mit den Standard-Groups:

  • Pelvis
  • Torso, Spine, Head
  • L_Arm, R_Arm + Hands
  • L_Leg, R_Leg + Feet
  • Optional Custom-spezifische Tracks für non-humanoide Elemente — Tail, Wings, Extra Arms, etc.

Bei AI Inbetweening Tutorial wurde gezeigt: ältere Cascadeur-Versionen (vor 2025.3) hatten alle Bones auf einem Track. Quick Rigging Tool in neueren Versionen separiert Pelvis automatisch. Falls dein Custom-Charakter in der älteren Struktur kommt: Quick Rigging Tool nochmal mit „Regenerate Rig" — die neue Struktur wird angelegt.

Inbetweening braucht Pelvis-Separation

Das ist der Hauptgrund für das Track-Hierarchy-Update: Inbetweening kann den Lower-Body separat vom Upper-Body generieren, wenn Pelvis und Beine in eigener Track-Gruppe sind. Bei nicht-separierter Hierarchie greift Inbetweening auf den ganzen Body — was bei sehr komplexen Custom-Charakteren zu unerwarteten Ergebnissen führen kann.

17 UE5-Integration des Custom-Charakters

FBX in UE5 importieren. UE5 fragt nach Skeleton — entweder ein bestehendes Skeleton verwenden (falls schon ein Custom-Skelett-Asset existiert) oder ein neues anlegen.

Bei Import werden angelegt:

  • Skeletal Mesh (Custom-Charakter)
  • Skeleton (Custom-Skelett)
  • Animation Sequence (die übertragene Animation)
  • Physics Asset (Default)

Im Sequencer oder Animation Blueprint kann der Custom-Charakter jetzt wie der UE5-Manny verwendet werden. Mehrere Animationen können auf dem Skeleton aufgenommen werden (Walk, Run, Sit, Gestik etc.) — jede ist eine eigene Animation Sequence.

Für Pipeline-Komplettheit: die gesamte Animation-Bibliothek aus 01-09 kann via Retargeting auf den Custom-Charakter übertragen werden. Statt nur eine Walk-Animation hast du dann eine komplette Bewegungs-Bibliothek für den Custom-Charakter.

Validierung Phase F

In UE5: Custom-Charakter steht im Level, mit übertragener Animation. Bewegung sieht plausibel aus (kleinere Proportions-Artefakte sind okay nach Phase E Polish). Skeleton ist sauber importiert. Animation Sequence kann im Animation Blueprint verwendet werden für Game-Logic oder im Sequencer für Cutscenes. Pipeline ist komplett geschlossen — von Blender-Model bis UE5-Animation.

Projekt 11+ (Kombinations-Szenen) ist der finale Akt der Reihe. Nicht mehr „lerne ein Werkzeug" oder „lerne einen Workflow" — sondern „kombiniere alles, was du gelernt hast, zu einer narrativen Sequenz". Charakter geht in einen Raum (Walk aus 03), bleibt stehen (Idle aus 01), gestikuliert beim Sprechen (08 + MetaHuman), setzt sich (04), steht auf (05), rennt raus (06), springt über ein Hindernis (07), landet, läuft aus. Optional auf einem Custom-Charakter (10) statt UE5-Manny.

Was du in dieser Reihe gelernt hast, ist nicht „Cascadeur in der Tiefe" — sondern „Cascadeur in deiner Pipeline". Das Werkzeug existiert nicht isoliert. Es kommt in einem konkreten Kontext zum Einsatz: Blender für Modeling, Auto-Rig Pro für Rigging, QuickMagic für Mocap, MetaHuman + FaceIt für Gesicht, Cascadeur für Body, UE5 für Komposition. Die Reihe hat versucht, in jeder Brochüre diesen Pipeline-Kontext mitzudenken.

Was Projekt 10 nicht macht

Kein non-humanoides Rigging (vierbeiniger Charakter, Mech, Insekt — alle anders). Kein Cloth-Sim oder Hair-Sim (separate UE5-Systeme). Kein Custom-Bones-Cascadeur-Tutorial (z.B. wenn der Charakter einen Schwanz hat — Cascadeur unterstützt das via Box Controller, aber das ist eine eigene Tiefe). Kein Multi-Charakter-Setup in einer Scene (das kommt in Projekt 11+).

Es gibt eine Faustregel in 3D-Pipelines: jedes Layer zwischen Charakter und Animation ist ein Engpass. Modelling → Rigging → Animation → Engine. Bei perfektem Workflow können diese Layer parallel laufen — Modelling-Update propagiert automatisch in Rigging, Rigging-Update propagiert in Animation, Animation in Engine. In der Praxis brechen die Layer ständig — neue Bone-Namen, Naming-Inconsistencies, FBX-Probleme. Pipeline-Engineer ist eine eigene Rolle in größeren Studios, deren Job nur darin besteht, diese Brüche zu reparieren. Was du in dieser Brochüre gemacht hast, ist die Pipeline-Bridge zwischen drei Tools (Blender, Cascadeur, UE5) und zwei Rig-Layern (Auto-Rig Pro Control-Rig, Game-Rig). Wenn du diesen Workflow einmal sauber im Kopf hast, kannst du ihn auch bei anderen Tools wiederholen — z.B. Maya → MotionBuilder → Unity, oder Houdini → Cascadeur → UE5.
